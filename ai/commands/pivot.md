# Command: Pivot (Course Correction)

**Focus:** Immediate redirection of the agent's current strategy or technical direction.

## Trigger
Use this command when the agent is:
- Over-engineering a solution.
- Using an incorrect library, pattern, or tech stack.
- Misunderstanding the core requirement or architectural intent.

## Steps
1. **Identify the Drift**: Explicitly state the current incorrect path or misunderstanding.
2. **Define the Correction**: Provide the new technical direction, constraint, or requirement.
3. **Synchronize Documentation**: Update the project's roadmap, task list, or design docs to reflect the new direction.
4. **Memory Log**: Ensure the reason for the pivot is recorded in the project's memory or session history to prevent future drift.

## Quality Bar
- The agent must summarize the *new* strategy and obtain user confirmation before proceeding.
- The pivot must be clearly documented to provide context for future development turns.
